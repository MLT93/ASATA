Modelo Vista Controlador

  1. model.php
  2. videogamesControler.php
  3. home.php
  4. routes.php

En index (es conveniente llamarlo así) tengo que llamar a las rutas que van a dirigir el tráfico a un lugar o a otro, es el punto de entrada de la aplicación

Explicación de la secuencia de funcionamiento:

  1. Introduzco la URL y el index obtiene el valor de $url
  2. La clase Router va a chequear la ruta y escoge (switch), creo instancia del controlador y escoge el método
  3. El método de ese controlador va a establecer la conexión con el modelo y va a enviar esa información a la vista

El archivo index coge la url que le pasamos a través de un parámetro. La dirección en este ejemplo es localhost/PHP/MVC?url="clientes". Recuerda que los parámetros son ?url="clientes; localhost/PHP/MVC(directorio raíz). Es como si se cargara todo a través de index.
